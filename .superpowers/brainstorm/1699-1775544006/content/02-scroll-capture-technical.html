Section 2: Scroll Capture — Technical Design

How frames are captured, compared, and stitched into a long image

Frame Capture Pipeline

How It Works Under the Hood
CAPTURE
DXGI Desktop Duplication API
Uses GPU-accelerated screen capture (IDXGIOutputDuplication).
Captures only the selected rectangle, not the full screen.
Runs on a background thread, triggered by scroll events.
Capture rate: On each scroll delta change — not a fixed timer. This ensures we capture every intermediate state.
DETECT
Overlap Detection via Column/Row Matching
For vertical scroll: compare bottom rows of previous frame to top rows of new frame.
For horizontal scroll: compare right columns of previous frame to left columns of new frame.
Algorithm: Compute a hash (or pixel-sum signature) for each row/column. Slide the new frame against the old to find best overlap match. Use Sum of Absolute Differences (SAD) on candidate rows to confirm alignment.
No overlap found? Concatenate directly — "don't miss content" is the priority.
STITCH
Incremental Strip Building
Once overlap is detected, append only the new (non-overlapping) portion to a growing bitmap strip.
The strip is stored in memory as a list of frame segments, not one massive bitmap — final composition happens at save time.
A downscaled thumbnail of the strip is rendered in real-time for the live preview panel.

Key Technical Decisions

Design Choices
Screen Capture API DXGI Desktop Duplication — GPU-accelerated, captures at display refresh rate, minimal CPU overhead. Falls back to BitBlt (GDI) for compatibility with older systems.
Direction Detection Determined by the first scroll input. Mouse wheel delta Y → vertical. Shift+wheel or horizontal trackpad → horizontal. Locked after first scroll event to prevent confusion.
Frame Storage Each unique strip segment stored as a separate bitmap in a list. Only composited into one large image at save time. This keeps memory bounded — we store only the non-overlapping deltas.
Duplicate Detection If a new frame is identical (or near-identical) to the previous frame, skip it. This handles pauses in scrolling without wasting memory.
Overlay Window Transparent, topmost WPF window covering all monitors. Click-through except in the selection area. Uses WS_EX_TRANSPARENT + WS_EX_TOOLWINDOW to avoid capturing itself.
